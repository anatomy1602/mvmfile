> 基于 ChatGPT 对 v0.1 的深度分析，结合项目实际状态，规划 v0.2 的核心改进方向。

## 二、外部诊断：v0.1 的四个结构性问题

### 问题 1：sections 是扁平列表，不是树结构

```
当前 v0.1：
sections: [section1, section2, section3, ...]
→ 扁平数组，无法表达嵌套关系

问题：
  - 不能把 quiz 放进 tab 里
  - 不能把 accordion 放进 timeline 里
  - 不能做任意嵌套布局
```

### 问题 2：没有素材引用能力

```
当前 v0.1：
  - 不支持图片
  - 不支持音频
  - 不支持视频
  - 不支持任意 URL 资源

问题：
  - 只能做纯文字内容
  - 无法用于视觉小说、课件、产品展示等场景
```

### 问题 3：渲染器不可扩展

```
当前 v0.1：
  InteractiveWebRenderer 里硬编码了所有组件
  加一个新组件 = 改渲染器源码

问题：
  - 无插件系统
  - 第三方无法扩展
  - 组件能力被锁定在渲染器内部
```

### 问题 4：没有能力声明机制

```
当前 v0.1：
  渲染器不知道自己需要支持什么
  .mvm 文件也不知道自己依赖什么能力

问题：
  - 跨端渲染时无法降级
  - PDF 渲染器遇到 quiz 怎么办？不知道
  - CLI 渲染器遇到 timeline 怎么办？不知道
```

## 三、v0.2 改进方向

### 改进 1：从扁平列表 → 树结构（P0，最重要）

**目标：** sections 支持嵌套，表达能力质变。

**当前语法（v0.1）：**
```yaml
sections:
  ## heading: 标题
  format: tab
  body: 内容
```

**改进后语法（v0.2）：**
```yaml
sections:
  ## heading: 前端框架对比
  format: tab
  children:
    ## heading: React
    body: Meta 开发的 UI 库...
    ## heading: Vue
    body: 渐进式框架...
    ## heading: Svelte
    body: 编译型框架...
```

**影响范围：**

| 模块 | 改动 |
|------|------|
| ModelParser | `Section` 类型新增 `children: array<Section>?` 字段 |
| MsgParser | 支持缩进嵌套解析 |
| 所有渲染器 | 递归渲染 children |
| CLI | 无需改动 |

**向后兼容：** 没有 `children` 的 section 行为与 v0.1 完全一致。

**分组机制简化：** 有了 children，`tab_group`、`accordion_group` 等分组字段可以废弃。分组关系由父子结构天然表达。

```
v0.1 方式（扁平 + group 字段）：
  section { format: tab, tab_group: "frameworks" }
  section { format: tab, tab_group: "frameworks" }
  section { format: tab, tab_group: "frameworks" }

v0.2 方式（树结构）：
  section { format: tab }
    children:
      section { heading: "React", body: "..." }
      section { heading: "Vue", body: "..." }
      section { heading: "Svelte", body: "..." }
```

### 改进 2：素材引用能力（P0）

**目标：** Model 层扩展 url/image/audio/video 类型，Msg 层可引用外部资源。

**MODEL 区扩展：**
```yaml
@type Section
heading: string
body: text
format: enum(paragraph|scene|...)
image: url?          # 图片素材
audio: url?          # 音频素材
video: url?          # 视频素材
background: url?     # 背景图
children: array<Section>?
```

**MSG 区使用：**
```yaml
sections:
  ## heading: 异世界的入口
  format: scene
  background: ./assets/forest.jpg
  image: ./assets/alice.png
  audio: ./assets/bgm_adventure.mp3
  body: |
    爱丽丝站在森林入口，深吸一口气。
```

**路径解析策略：**

| 路径格式 | 解析方式 | 示例 |
|----------|---------|------|
| `./xxx` | 相对于 .mvm 文件 | `./assets/bg.jpg` |
| `xxx` | 约定目录查找 | `forest.jpg` → `./assets/background/forest.jpg` |
| `https://` | 直接引用 | `https://example.com/img.jpg` |

**渲染器行为：** 素材不存在时降级为占位符 + 控制台警告，不中断渲染。

### 改进 3：插件化渲染协议（P1）

**目标：** 渲染器支持组件插件，第三方可扩展。

**架构变更：**
```
v0.1：
  InteractiveWebRenderer 内部硬编码所有组件

v0.2：
  InteractiveWebRenderer
    ├── 内置组件（tab, accordion, quiz, timeline, flashcard, reveal）
    └── 插件目录（plugins/）
        ├── my_custom_component.py
        └── another_plugin.py
```

**插件接口：**
```python
class ComponentPlugin:
    name: str                    # 组件名（对应 format 枚举值）
    def render(self, section, children_html: str) -> str:
        """返回该组件的 HTML 字符串"""
        ...
    def get_css(self) -> str:
        """返回组件所需的 CSS"""
        ...
    def get_js(self) -> str:
        """返回组件所需的 JavaScript"""
        ...
```

**.mvm 文件中声明插件依赖：**
```yaml
@plugins
- custom_chart
- data_table
```

**渲染器加载流程：**
```
1. 解析 @plugins 声明
2. 查找内置组件 → 找到则使用
3. 查找插件目录 → 找到则加载
4. 都没找到 → 降级为 paragraph + 警告
```

### 改进 4：能力声明系统（P1）

**目标：** 渲染器声明自己支持的能力，.mvm 文件声明自己需要的能力，不匹配时优雅降级。

**渲染器侧：**
```python
class WebRenderer(BaseRenderer):
    capabilities = {"tab", "accordion", "quiz", "timeline",
                    "flashcard", "reveal", "image", "audio", "video"}

class MdRenderer(BaseRenderer):
    capabilities = {"paragraph", "quote", "code", "list", "image"}
    # 不支持交互组件，遇到时降级
```

**.mvm 文件侧（可选声明）：**
```yaml
@requires
- tab
- quiz
- image
```

**降级策略：**
```
渲染器遇到不支持的 format：
  1. 有 children → 递归渲染 children，忽略容器交互
  2. 有 image → 渲染为 [图片: path] 文字链接
  3. 有 audio/video → 渲染为 [媒体: path] 文字链接
  4. 纯交互（quiz/flashcard）→ 降级为 paragraph
```

## 四、v0.2 任务清单

### 阶段 A：树结构与素材引用（核心）

- [ ] **A.1 Schema 扩展**
  - Section 类型新增 `children: array<Section>?`
  - Section 类型新增 `image: url?`、`audio: url?`、`video: url?`、`background: url?`
  - url 类型支持相对路径、约定目录、URL 三种格式

- [ ] **A.2 MsgParser 嵌套解析**
  - 支持缩进层级解析 children
  - 向后兼容：无缩进时行为与 v0.1 一致
  - 嵌套深度限制（建议最大 5 层）

- [ ] **A.3 渲染器递归改造**
  - WebRenderer：递归渲染 children
  - MdRenderer：递归渲染 children（嵌套列表）
  - SlideRenderer：递归渲染 children（嵌套幻灯片）
  - InteractiveWebRenderer：递归渲染 children（嵌套交互组件）

- [ ] **A.4 素材渲染支持**
  - WebRenderer：`<img>` / `<audio>` / `<video>` 标签
  - InteractiveWebRenderer：scene 格式（背景图 + 立绘 + 音频）
  - MdRenderer：`![alt](path)` / `[音频](path)` 链接
  - 路径解析器：相对路径 / 约定目录 / URL

- [ ] **A.5 示例文件更新**
  - 更新 interactive-demo.mvm 使用树结构语法
  - 新增素材引用示例
  - 验证向后兼容（v0.1 格式仍可解析）

### 阶段 B：插件化与能力声明

- [ ] **B.1 组件插件接口**
  - 定义 ComponentPlugin 基类
  - 实现 render / get_css / get_js 接口
  - 插件发现与加载机制

- [ ] **B.2 @plugins 声明解析**
  - ModelParser 支持 @plugins 指令
  - 插件目录约定（./plugins/ 或 .mvm 同级）

- [ ] **B.3 能力声明系统**
  - BaseRenderer 新增 capabilities 属性
  - 渲染前检查能力匹配
  - 不匹配时降级策略

- [ ] **B.4 @requires 声明解析**
  - ModelParser 支持 @requires 指令
  - CLI validate 命令检查能力匹配
  - 不匹配时输出警告

### 阶段 C：v0.2 验证与发布

- [ ] **C.1 向后兼容测试**
  - v0.1 格式的 .mvm 文件在 v0.2 解析器中正常工作
  - v0.1 渲染器输出与 v0.2 一致（无 children 时）

- [ ] **C.2 新功能演示**
  - 树结构嵌套交互演示
  - 素材引用演示（图片 + 音频）
  - 插件组件演示

- [ ] **C.3 文档更新**

## 五、不做的事（v0.2 红线）

| 不做 | 原因 |
|------|------|
| 条件分支 / if-else | 属于应用层，不是内容层 |
| 状态管理 / 好感度 | 属于应用层 |
| 存档 / 读档 | 属于应用层 |
| 数据绑定 / for 循环 | 复杂度过高，留到 v0.3 |
| 可视化编辑器 | 工具链问题，留到后续阶段 |
| 编译为静态站点 | 优化问题，留到后续阶段 |

> **MVM 是内容格式，不是应用框架。**
> v0.2 的目标是让内容层能力更完整，而不是让 MVM 变成应用引擎。

---

## 六、v0.2 与 v0.1 的关系

```
v0.1 = "带类型系统 + 交互声明"的 Markdown++
v0.2 = "带树结构 + 素材引用 + 插件系统"的内容 IR

v0.2 不是推翻 v0.1，而是在 v0.1 基础上补齐三个关键短板：
  1. 结构能力：扁平 → 树
  2. 素材能力：纯文字 → 多媒体
  3. 扩展能力：硬编码 → 插件化
```

## 七、技术选型补充

| 模块 | 选定方案 | 备注 |
|------|---------|------|
| 树结构解析 | 缩进层级 + 递归 | 类 Python/YAML 风格 |
| 素材路径 | pathlib 相对路径 | 跨平台兼容 |
| 插件系统 | Python entry_point 风格 | 简单目录扫描 |
| 能力声明 | Python set 集合 | O(1) 查找 |
| 降级策略 | 策略模式 | 每种 format 有默认降级方案 |
